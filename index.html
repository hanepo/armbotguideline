Robot Arm Color Sorter

Automatically detects red green blue objects and sorts them into matching containers — powered by Raspberry Pi 4 and computer vision.

Live animation — watch the arm detect color and sort automatically

★How It Works

A camera mounted overhead watches a 20×20cm work zone divided into 9 areas. When a colored box is placed anywhere in the zone, the system:

  1. Detects the box color and position using computer vision (OpenCV)
  2. Rotates the arm to that exact zone
  3. Closes the claw to grab the box
  4. Moves to the matching container (red → red box, etc.)
  5. Releases and returns to home position

💻Connect to the Raspberry Pi

You control the Pi remotely from your laptop using SSH — no monitor needed.

Requirements

  • Your laptop must be on the same WiFi as the Pi
  • Mac/Linux: use Terminal  |  Windows: use PowerShell or PuTTY

SSH command

ssh pi@robotarm.local
USERNAME

First time connecting? It will ask: "Are you sure you want to continue connecting?" — type yes and press Enter. Only asks once.

If robotarm.local doesn't work — 3 ways to find the IP

Method 1 — Easiest: Use the Fing app (recommended)
  1. Download Fing app on your phone (free, App Store or Google Play)
  2. Make sure your phone is on the same WiFi as the Pi
  3. Open Fing → tap Scan for devices
  4. Look for a device named robotarm or Raspberry Pi
  5. The number next to it (e.g. 192.168.1.45) is the IP address
ssh pi@192.168.1.45   ← use YOUR actual number from Fing
Method 2 — From Mac/Linux Terminal

Open Terminal and type:

ping robotarm.local

You will see something like:

PING robotarm.local (192.168.1.45): 56 data bytes

The number in brackets (192.168.1.45) is the IP. Press Ctrl+C to stop, then:

ssh pi@192.168.1.45
Method 3 — Router admin page
  1. Open any browser on your laptop
  2. Type 192.168.1.1 in the address bar and press Enter
    (if that doesn't work, try 192.168.0.1 or 10.0.0.1)
  3. Login — username and password are usually printed on a sticker on the bottom of your router
  4. Look for a section called Connected Devices, DHCP Clients, or Device List
  5. Find robotarm in the list — the number next to it is the IP address
  6. Use that IP: ssh pi@192.168.x.x
Tip: The IP address may change every time the Pi reboots. Method 1 (Fing) is the most reliable way to find it each time.

1Wiring the Servos

Each servo has 3 wires: brown (ground), red (power), orange (signal). Power comes from the PSU — never from the Pi GPIO pins.

⚠ Warning: Connect servo red (power) wires to the PSU breadboard rail ONLY. Connecting them to Pi GPIO pins will overheat and melt the wires.

2Software Setup

SSH into the Pi, then run these commands in order:

1. Update the system

sudo apt update && sudo apt upgrade -y

2. Install pigpio (servo control)

sudo apt install git -y
git clone https://github.com/joan2937/pigpio.git
cd pigpio && make && sudo make install
sudo cp /home/pi/pigpio/util/pigpiod.service /lib/systemd/system/
sudo sed -i 's|/usr/bin/pigpiod|/usr/local/bin/pigpiod|' /lib/systemd/system/pigpiod.service
sudo systemctl daemon-reload
sudo systemctl enable pigpiod
sudo systemctl start pigpiod

3. Install Python libraries

sudo apt install python3-opencv rpicam-apps python3-picamera2 -y
pip3 install flask --break-system-packages

4. Copy the project files

git clone https://github.com/hanepo/armbotguideline.git ~/armbot
cd ~/armbot
Verify: Run sudo systemctl status pigpiod — should say active (running) in green.

3Camera Setup

The Pi Camera Rev 1.3 connects via ribbon cable to the Pi's CSI port.

  • Lift the small plastic clip on the Pi's CSI port (between USB-C and HDMI)
  • Slide the ribbon cable in — blue strip faces toward the HDMI port
  • Push the clip down to lock it
  • Mount camera directly overhead, centered above the arm base, pointing straight down
  • Use a 50cm ribbon cable extension to reach the overhead position

Test the camera

rpicam-jpeg -o test.jpg

Should print: Still capture image received

Preview + grid alignment

cd ~/armbot && python3 camera_test.py

Open http://<pi-ip>:5001 on any browser. Use the sliders to align the green grid with the masking tape on the table.

Each zone is approximately 7×7 cm. All 9 zones are valid pickup positions.

Run the calibration tool

python3 calibrate.py

For each zone (1–4, 6–9) and each drop container:

  1. Place a box in the zone shown on screen
  2. Use ← → to select a servo, ↑ ↓ to move it
  3. Move the arm until the claw is directly above the box
  4. Press S to save, then N for the next position
Tip: Saved positions are stored in positions.json. You can re-run calibrate.py any time to adjust.

5Run the Sorter

Start the program

cd ~/armbot && python3 sorter.py

The arm moves to home position and the camera starts watching the grid. Place any colored box anywhere in zones 1–4 or 6–9 and the arm will:

  1. Detect the box color and which zone it's in
  2. Rotate and reach to that zone
  3. Grab the box with the claw
  4. Move to the matching container
  5. Drop the box and return home
Colors supported: Red Green Blue

Stop the program

Press Ctrl + C

6Troubleshooting

Servo not moving

Check: PSU is on (green LED) · Shared ground wire from Pi GND to breadboard blue rail · sudo systemctl status pigpiod shows running

Camera not detected

Check: Ribbon cable is fully seated · Blue strip faces HDMI · Run rpicam-jpeg -o test.jpg to verify

Wrong color detected

Ensure good lighting (no direct light into lens) · Use solid-colored objects · Adjust HSV thresholds in sorter.py if needed

Arm misses the box

Re-run python3 calibrate.py and adjust the zone positions · Make sure box is fully inside the zone square
